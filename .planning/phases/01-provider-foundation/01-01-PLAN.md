<task type="auto">
  <name>Task 1: Create Provider ABC and ProviderManager</name>
  <files>gateway/providers/__init__.py, gateway/providers/base.py</files>
  <action>
Create the gateway/providers/ directory structure.

**gateway/providers/base.py:**
Create Provider abstract base class following the pattern from clara_core/platform.py but designed for gateway-embedded providers (not WebSocket clients).

Required interface:
- `name` property (abstract) - Provider identifier (e.g., "discord", "email")
- `async start()` (abstract) - Start the provider (initialize bot, connect)
- `async stop()` (abstract) - Stop the provider (cleanup, disconnect)
- `normalize_message(platform_msg: Any) -> PlatformMessage` (abstract) - Convert platform-specific message
- `async send_response(context: dict, content: str, files: list[str] | None = None)` (abstract) - Send response

Add a `running` property that tracks provider state.

**gateway/providers/__init__.py:**
Create ProviderManager singleton following the pattern from clara_core/mcp/manager.py.

Required methods:
- `get_instance()` class method - Returns singleton
- `register(provider: Provider)` - Add a provider to registry
- `unregister(name: str)` - Remove a provider
- `get(name: str) -> Provider | None` - Get provider by name
- `async start_all()` - Start all registered providers
- `async stop_all()` - Stop all providers
- `providers` property - Dict of name -> Provider

Export convenience function `get_provider_manager()`.

Use asyncio.Lock for thread safety on start/stop operations.
